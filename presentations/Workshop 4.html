[ PART 1 · DEF ]
04

Define it once.

def, a name, parentheses, a colon — then an indented body. Calling the name runs the body.

def greet(): print("Hello!") print("Welcome to the club.") greet()
[ Output ]
Hello!
Welcome to the club.

Function names follow the same snake_case rules as variables (Workshop 02), and the colon + indented block works exactly like if and while from Workshop 03. The only new word is def.

[ PART 1 · CALLING ]
05

def alone does nothing.

def only stocks the vending machine. Nothing happens until someone presses the button — and every press replays the body.

def greet(): print("Hello!") print("Before any call") greet() greet()
[ Output ]
Before any call · Hello! · Hello!

Python reads the def, memorizes the recipe, and moves on — that’s why Before any call prints first. And define before you call: scripts still run top to bottom (Workshop 01).

[ PART 1 · PARAMETERS ]
06

Leave a slot open.

name is a blank inside the recipe — a coin slot on the machine. Each call fills it with a different value.

def greet(name): print(f"Hello, {name}!") greet("nino") greet("giorgi")
[ Output ]
Hello, nino!
Hello, giorgi!

One function, every guest. This is the fix for slide 03 — change the greeting once, and every call gets the new version.

[ PART 1 · MULTIPLE PARAMETERS ]
08

More slots, in order.

Separate parameters with commas. Python matches by position: first argument → first parameter.

def order(drink, size): print(f"One {size} {drink}, coming up!") order("latte", "large") order("espresso", "small") order("small", "espresso") # oops — swapped!
[ Output ]
One large latte, coming up! · One small espresso, coming up!
One espresso small, coming up!

Python doesn’t know what a latte is. It only knows first value → first slot. One espresso small is your bug, not Python’s.

[ PART 1 · DEFAULTS ]
09

A fallback value.

Put = in the definition and the slot fills itself when the caller stays silent.

def greet(name="friend"): print(f"Hello, {name}!") greet("nino") greet()
[ Output ]
Hello, nino!
Hello, friend!

Pass an argument and it wins; pass nothing and the default steps in. Defaults make arguments optional.

[ PART 2 · RETURN ]
12

Hand the answer back.

So far our functions only talked at the screen. return is the can dropping out of the machine — a value handed back to whoever called.

def add(a, b): return a + b result = add(3, 4) print(result) print(add(10, 5))
[ Output ]
7
15

add(3, 4) doesn’t print 7 — it becomes 7, right there in your code. Catch it in a variable, or use it on the spot.

A call is a value.

Anywhere a number can go, a call can go — arithmetic, f-strings, if conditions.

def area(width, height): return width * height total = area(3, 4) + area(5, 2) print(f"{total = }") if area(10, 10) > 50: print("Big room!")
[ Output ]

[ PART 2 · PRINT VS RETURN ]
15

Print is not return.

print — the display lights up
def show_sum(a, b): print(a + b) show_sum(2, 3) # 5 flashes on screen — then it's gone
return — the can drops out
def add(a, b): return a + b total = add(2, 3) print(total + 10) # 15 — the value kept working

[ The trap ]print() shows a value to a human and throws it away. return hands the value back to your code. A vending machine test: print is the little screen flashing — return is the can in your hand. You can’t drink the screen.

[ PART 2 · NONE ]
16

The sound of nothing.

So what do you get if you catch the result of a function that only prints?

def show_sum(a, b): print(a + b) result = show_sum(2, 3) print(f"{result = }")
[ Output ]
5
result = None

Every function returns something. No return line means Python quietly returns None — its word for “no value here.” If a variable mysteriously prints None, the function behind it printed where it should have returned.

[ PART 2 · RETURN EXITS ]
17

Return ends it. Instantly.

The moment return runs, the function is over — code after it never executes.

def check(n): if n % 2 == 0: return "even" return "odd" print(check(10)) print(check(7))
[ Output ]
even · odd

That’s why the last line needs no else — reaching it proves the if didn’t fire. The % trick is from Workshop 01; the early exit is new.

[ PART 2 · SCOPE ]
18

Variables stay inside.

total is born when the call starts and vanishes when it ends. Outside the function, it was never defined.

def make_total(price, quantity): total = price * quantity return total print(make_total(7, 3)) print(total) # NameError!
[ Output ]
21
NameError: name 'total' is not defined

That’s a feature, not a bug — functions can’t trample each other’s variables. The only door out is return.

[ PART 2 · CAPSTONE ]
19

Grades, on an assembly line.

Workshop 03’s grade ladder, rebuilt with early returns — then fed by a Workshop 03 for loop.

the machine — one definition
def letter_grade(score): if score >= 90: return "A" if score >= 80: return "B" if score >= 70: return "C" return "F"
the assembly line — four calls
for score in range(65, 100, 10): grade = letter_grade(score) print(score, "->", grade) # 65 -> F # 75 -> C # 85 -> B # 95 -> A

One definition, four calls, zero copy-paste — def + if + for + range(start, stop, step), everything from Workshops 1–3 in a dozen lines. This is why functions exist.
