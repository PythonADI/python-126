[ PART 1 · VARIABLES ]
03

A variable is
a label.

You give a value a name, then use that name anywhere you need the value. Change it once — it updates everywhere.

message = "Hello, world!" print(message)
[ Output ]
Hello, world!
Python · Workshop 02Variables
[ PART 1 · NAMING ]
04

Name things
clearly.

Letters, numbers and underscores only; no spaces; never start with a number. Then just be descriptive.

[ Good ]
  • student_name
  • score_1
  • total_points
[ Avoid ]
  • 1_score — starts with a number
  • student name — has a space
  • print — a Python word

Misspell a name later and Python stops with a NameError — it only knows the names you defined.

Python · Workshop 02Naming
[ PART 1 · STRINGS ]
05

Text is a
string.

Anything in quotes — single or double. Python can reshape it for you with simple methods.

name = "ada lovelace" print(name.title()) # Ada Lovelace print(name.upper()) # ADA LOVELACE

A method is an action attached with a dot: .title(), .upper(), .lower().

Python · Workshop 02Strings
[ PART 1 · F-STRINGS ]
06

Drop values
into text.

Put an f before the quotes, then wrap any variable in { }. Python fills in the blanks.

first = "ada" last = "lovelace" print(f"Hello, {first.title()} {last.title()}!")
[ Output ]
Hello, Ada Lovelace!

[ PART 1 · WHITESPACE ]
07

Spacing you
can’t see.

Add a tab with \t and a new line with \n. Trim stray spaces from the edges with .strip().

print("Languages:\n\tPython\n\tRust") name = " ada " print(name.strip()) # "ada"

[ PART 1 · NUMBERS ]
08

Python does
the math.

Whole numbers (int) and decimals (float) work like a calculator. ** means “to the power of”.

print(2 + 3 * 4) # 14 print(3 ** 2) # 9 print(0.5 * 2) # 1.0 big = 14_000_000_000

Underscores make big numbers readable — Python ignores them.

[ THE PROBLEM ]
12

It worked
yesterday.

You change one thing — and everything breaks. You can't remember what the working version looked like. So you start saving copies… and it only gets worse.

[ Without git ]
  • project.py
  • project_final.py
  • project_final_v2.py
  • project_FINAL_real.py
  • …which one actually worked?
[ With git ]
  • One file. One clean folder.
  • Every version is remembered
  • Jump back to any point in time
  • Nothing is ever truly lost

[ THE TOOL ]
14

Git takes the
snapshots.

Git is the free tool that does version control on your computer. It's the industry standard — used by nearly every software team on earth — and it quietly remembers every version you save.

You talk to it with short commands in the terminal. Don't worry — you'll mostly reuse just a handful.

Python · Workshop 02Meet git
[ THE BIG ONE ]
15

Git is not GitHub.

Git is the camera that takes the snapshots. GitHub is the album where you store and share them online.

[ Git ]
  • A tool on your computer
  • Takes & stores the snapshots
  • Works fully offline
  • Built in 2005 by Linus Torvalds
[ GitHub ]
  • A website, in the cloud
  • Keeps a copy of your project online
  • Backup, sharing & teamwork
  • One of many hosts (also GitLab)

[ FIRST COMMANDS ]
20

Your first repository.

git init git status git add . git commit -m "First snapshot" git log

One-time setup first: git config --global user.name "Luka" and your email, so every snapshot is signed. git add . means "all files"; the text after -m is a note to future-you.

[ THE PAYOFF ]
23

Broke it all?
Go back.

git checkout . git checkout 9f3c2a git checkout main

Because every snapshot is saved, a mistake is never permanent. git checkout . discards unsaved chaos; git checkout <id> rewinds to an old commit; git checkout main brings you back. Worst case: you lose a few minutes — not your project.
