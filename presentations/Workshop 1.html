[ A REAL PROBLEM ]
06

How does a map find
the fastest route?

Your taxi app. A game character chasing you. A delivery robot. They all solve the same puzzle: get from A to B, the smart way.

Let's see how a computer actually looks at a map.

Python · Workshop 01Pathfinding
[ STEP 01 · THE MAP ]
07

A grid. Black cells are walls you can't cross. The goal: a path from [S] to [G].

Python · Workshop 01The map
[ STEP 02 · POINTS ]
08

To a computer, the map is just points. Every open cell becomes one — a [vertex].

Python · Workshop 01Vertices
[ STEP 03 · CONNECTIONS ]
09

Each point links to its neighbours. A connection is an [edge]. Together they form a graph — a web of places and paths.

Python · Workshop 01Edges
[ VOCABULARY ]
10

[VERTEX] — a place you can be: a junction, a tile, a city.

[EDGE] — a path between two places, carrying a cost: distance, time, or fuel.

Find a route = pick edges from start to goal so the total cost is as small as possible.

Python · Workshop 01Vertex / Edge
[ STEP 04 · A* (A-STAR) ]
11

Watch it think.

▶ Open the A* video — Sebastian Lague (E01)

A* explores toward the goal first, using a smart guess of the distance left. So it finds the shortest path without checking everything.

The whole algorithm? A few dozen lines of Python.

The same idea routes your taxi, moves game enemies, and steers robots.
