[ PART 1 · IF ]
04

Your first
decision.

if runs its block only when the condition is True. The colon opens the block.

age = 19 if age >= 18: print("You can vote.") print("Done.")

Forget the : and Python stops with a SyntaxError before it runs a single line.

Python · Workshop 03if
[ PART 1 · INDENTATION ]
05

Indentation
is the block.

Python has no curly braces. The indentation itself marks what is inside the if — and what is outside.

if logged_in: print("Welcome!") # inside — runs only if True print("Loading…") # inside — runs only if True print("Footer") # outside — always runs

Four spaces is the convention. Being inconsistent — or mixing tabs and spaces — throws an IndentationError.

[ PART 1 · ELSE ]
06

The other
path.

else catches every case the if didn’t — it runs when the condition is False.

temp = 14 if temp >= 20: print("T-shirt weather.") else: print("Bring a jacket.")

Exactly one branch runs — never both, never neither.

Python · Workshop 03else
[ PART 1 · ELIF ]
07

Many paths.

Chain conditions with elif. Python checks top to bottom and takes the first that is True — then skips the rest.

score = 84 if score >= 90: grade = "A" elif score >= 80: grade = "B" elif score >= 70: grade = "C" else: grade = "F" print(grade) # B

[ PART 2 · THE PROBLEM ]
12

Don’t repeat
yourself.

By hand
print(1) print(2) print(3) print(4) print(5) # …and on, and on
With a loop
for n in range(1, 6): print(n)

Same output. One is busywork; the other scales to a thousand lines without changing.

Repeat while

A while loop keeps running its block as long as the condition stays True.

[ PART 2 · INFINITE LOOPS ]
14

The endless
loop.

If the condition never becomes False, the loop never stops. The classic cause: forgetting to change the variable.

n = 3 while n > 0: print(n) # forgot n -= 1 → n stays 3 forever

[ The trap ]This prints 3 forever. Every while loop needs something inside that moves it toward False. Press Ctrl + C to stop a runaway loop.

The counting
loop.

for with range() repeats a set number of times — no counter to manage by hand.

for i in range(5): print(i)

range(5) starts at 0 and stops before 5 — five numbers, not six.

[ PART 2 · RANGE ]
17

range(),
three ways.

range(5)
→
0 1 2 3 4stop
range(2, 6)
→
2 3 4 5start, stop
range(1, 10, 2)
→
1 3 5 7 9start, stop, step
range(10, 0, -1)
→
10 9 8 … 1count down

The stop value is never included — that’s why range(2, 6) ends at 5.

[ PART 2 · STRINGS ]
18

Loop over
letters.

Strings are sequences — a for loop walks them one character at a time.

for char in "Python": print(char)

Prints P · y · t · h · o · n — each on its own line. Same for, no range() needed.

[ PART 2 · STEERING ]
19

Steer the
loop.

break — stop entirely
for n in range(1, 99): if n == 5: break print(n) # 1 2 3 4
continue — skip one
for n in range(1, 6): if n == 3: continue print(n) # 1 2 4 5

break leaves the loop for good. continue jumps to the next turn.
