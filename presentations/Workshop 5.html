So far: one box,
one value.

A variable is a single box (Workshop 01). age = 21 holds one number; name = "nino" holds one word. But a shopping run has many items, a class has many scores, a door has many guests.

A list is one variable that holds many values, in order. And everything you already know — for, len, in, indexing — is about to pay off all at once.

[ PART 1 · THE PROBLEM ]
03

Five variables. One idea.

Storing five scores the only way you knew until today — five separate names:

score1 = 70 score2 = 85 score3 = 90 score4 = 60 score5 = 95

Now find the average. Or add a sixth. Or loop over them. You can’t — each name is its own island. Five values that belong together, with no thread tying them. A list is that thread: scores = [70, 85, 90, 60, 95].

[ PART 1 · CREATING ]
04

Square brackets, commas.

Wrap values in [ ], separate them with commas. That’s a list — and a list is itself one value you store in a variable.

shopping = ["milk", "bread", "eggs", "cheese"] numbers = [10, 20, 30, 40, 50] empty = [] print(shopping) print(len(shopping))
[ Output ]
['milk', 'bread', 'eggs', 'cheese']
4

len() — the same function that counted a string’s letters in Workshop 03 — counts a list’s items. And [] is an empty list, ready to be filled.

[ PART 1 · WHAT A LIST LOOKS LIKE ]
05

A row of numbered boxes.

shopping = ["milk", "bread", "eggs", "cheese"]

The value lives inside the box; the index is its address. So shopping[0] is "milk", shopping[-1] is "cheese", and len(shopping) is 4.

[ PART 1 · INDEXING ]
06

Counting starts at zero.

Every item has a position — its index. The first is [0], not [1]. Negative numbers count from the end.

shopping = ["milk", "bread", "eggs", "cheese"] print(shopping[0]) # milk - the first print(shopping[2]) # eggs print(shopping[-1]) # cheese - the last
[ Output ]
milk · eggs · cheese

[-1] is always the last item, however long the list grows — you never have to know the length to reach the end.

[ PART 1 · SLICING ]
08

Take a slice.

Two indexes and a colon — [start:stop] — give you a piece. The stop is not included. Leave a side blank to mean “the end.”

nums = [10, 20, 30, 40, 50] print(nums[1:4]) # [20, 30, 40] print(nums[:3]) # [10, 20, 30] print(nums[2:]) # [30, 40, 50]
[ Output ]
[20, 30, 40]
[10, 20, 30]
[30, 40, 50]

A slice is itself a list. And the very same [1:4] works on a string — they’re both sequences.

[ PART 1 · THE EDGE ]
09

Mind the edge.

Four items live at positions 0, 1, 2, 3. Ask for [4] and Python stops you cold.

shopping = ["milk", "bread", "eggs", "cheese"] print(shopping[4])
[ Output ]
IndexError: list index out of range

[ The classic off-by-one ]A list of n items has its last index at n - 1. Looping with range(len(x)) already gets this right — but range(len(x) + 1) walks one step off the cliff. Watch for it in this week’s bonus.

[ PART 1 · SEQUENCES ]
10

You’ve done this before.

A string is characters in a row; a list is values in a row. The same moves work on both.

a string
word = "python" word[0] # 'p' word[-1] # 'n' word[1:4] # 'yth' len(word) # 6 for ch in word: ...
a list
nums = [10, 20, 30] nums[0] # 10 nums[-1] # 30 nums[1:3] # [20, 30] len(nums) # 3 for n in nums: ...

Indexing, slicing, len, in, and for behave the same on both — they’re all sequences. Learn it once, use it everywhere.

[ PART 2 · LOOPING ]
12

One item at a time.

A for loop walks a list exactly like it walked a string’s characters in Workshop 03 — but now each step hands you a whole item.

shopping = ["milk", "bread", "eggs"] for item in shopping: print(item)
[ Output ]
milk · bread · eggs

Need the position too? Loop the indexes with for i in range(len(shopping)) and reach in with shopping[i] — that’s how you print a numbered list.

[ PART 2 · MEMBERSHIP ]
13

Is it in there?

in asks a yes/no question — a bool, perfect for an if. .count() tells you how many.

shopping = ["milk", "bread", "milk"] if "bread" in shopping: print("got it") print("eggs" not in shopping) # True print(shopping.count("milk")) # 2
[ Output ]
got it
True
2

You’ve used in as the engine of every for loop. On its own, beside an if, it’s a search.

[ PART 2 · THE CORE PATTERN ]
16

Start empty. Grow it.

The most common list pattern in all of programming: an empty list, a loop, and .append() on every pass.

evens = [] for n in range(1, 11): if n % 2 == 0: evens.append(n) print(evens)
[ Output ]
[2, 4, 6, 8, 10]

evens = [] goes before the loop, never inside — inside, you’d wipe it clean on every pass. Filtering, collecting input, transforming: it’s all this one shape.

[ PART 2 · LISTS + FUNCTIONS ]
17

Hand a list to a function.

A parameter can be a whole list. Last week’s def + return now chew through many values at once.

def average(numbers): total = 0 for n in numbers: total += n return total / len(numbers) print(average([70, 85, 90, 60]))
[ Output ]
76.25

Workshop 03 built the loop, Workshop 04 built the function — together they fold a whole list of scores into one answer you can reuse anywhere.

[ PART 2 · STRINGS & LISTS ]
18

Strings to lists, and back.

.split() breaks a string into a list of words; .join() glues a list back into one string.

sentence = "now this has words" words = sentence.split() print(words) # ['now', 'this', 'has', 'words'] print(len(words)) # 4 print("-".join(words)) # now-this-has-words
[ Output ]
['now', 'this', 'has', 'words']
4
now-this-has-words

Remember Workshop 03’s word counter, hunting for spaces one character at a time? len(text.split()) — that whole loop, in a single line.
